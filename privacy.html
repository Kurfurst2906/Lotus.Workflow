Privacy Policy for Lotus.Workflow

Last updated: 2026-02-09

1. Information We Access

Lotus.Workflow uses Google OAuth to access the following Google user data to function as an internal operational dashboard:

  • Google Sheets: Read and write access to specific spreadsheets used as the system's database (e.g., Workflow, Labor, and Master Data sheets).
  • Google Drive: Access to save generated reports (PDFs) and read configuration or template files.
  • Google User Identity: Basic profile information (email address and name) to identify the current user for authentication and role-based access control.
  • Gmail (if applicable): Permission to send automated notifications regarding workflow status changes or approval requests on your behalf.

2. How We Use the Information

The accessed data is used solely to:

  • Automate internal workflows, including routing requests and managing approvals.
  • Manage operational data such as labor, personnel, and project information.
  • Generate and export operational reports (e.g., PDF documents) to Google Drive.
  • Render visualizations such as organizational charts and data snapshots.
  • Maintain audit trails (e.g., “Approved by [User]”, “Created by [User]”).

3. Data Storage and Sharing

  • No External Storage: This application does not store data on external servers. All data remains within the organization's Google Workspace environment.
  • No Third-Party Sharing: Data is not shared, sold, or transferred to third parties, AI models, or advertising networks.

4. Data Security

All data access is performed through Google Apps Script and complies with Google OAuth security requirements. The system follows a layered architecture to ensure data integrity and security.

5. User Control

Users can revoke access to this application at any time via their Google Account settings (Security > Third-party apps with account access).
